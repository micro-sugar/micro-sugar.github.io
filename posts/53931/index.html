VSTS於Build和Release時會用到的位置變數

發表於 2018-09-10 | 分類於 VSTS |

剛開始用VSTS的時候常被他的Build或Release的位置變數搞混
所以這篇文章記錄一下自己常用的變數

下面有些資料夾可以用不同變數到同一個位置，我就放在一起了

需要完整的變數表可以從備註/相關資料連結去Microsoft的網站上查詢

Build

VSTS或TFS的Agent在執行Build的時候主要是在4個資料夾內進行動作

BuildDirectory

For example: c:\agent_work\1

  • 給該pipeline的資料夾
  • 有些套件會在這個地方增加資料夾給套件自己使用

ArtifactStagingDirectory

For example: c:\agent_work\1\a

  • Artifacts是您希望構建生成的文件(例如，您有一個C＃或C ++ .NET Windows應用程序的.DLL和.EXE可執行文件和.PDB符號文件。)
  • 換個方式說就是待部屬的資料夾
  • 此資料夾在每次新的Build之前會自動清除此目錄，因此不必自己清理它。

BinariesDirectory

For example: c:\agent_work\1\b

  • 預設不會自動清除，如須清除請於Get sources的Clean選True，Clean options選擇Sources and output directory(此選項會刪除BinariesDirectory，ArtifactStagingDirectory，TestResultsDirectory)
  • 由組建代理程式編譯的檔案，例如已編譯的應用程式檔案。

SourcesDirectory

For example: c:\agent_work\1\s

  • 由組建代理程式讀取的檔案，例如原始程式檔。
  • VSTS的Get sources會將Source下載至子資料夾
  • 預設是不會清除的

TestResults

For example: c:\agent_work\1\TestResults

  • 由組建代理程式執行的任何測試產生的檔案。

Release

Release的部分基本上只會產生一個資料夾

ArtifactsDirectory

For example: C:\agent_work\r1\a

備註

上述這些資料夾是指在預設的情況
在執行階段利用command line或是powershell可以做另外的操控(如上所述，部分擴充套件也會建立自己的資料夾喔)
